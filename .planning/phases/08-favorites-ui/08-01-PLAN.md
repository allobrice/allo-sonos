  <action>
Create `frontend/src/stores/favorites.ts` as a composition-style Pinia store (`defineStore('favorites', () => { ... })`), following the exact same pattern as `stores/zones.ts`.

**Exported type:**
```typescript
export interface Favorite {
  title: string
  type: string   // 'station' | 'playlist' | 'album' | 'other'
  uri: string
  albumArtURI: string | null
}
```

**State:**
- `favorites: ref<Favorite[]>([])` — cached list
- `loading: ref(false)` — true while fetching
- `error: ref<string | null>(null)` — error message or null

**Actions:**
- `loadFavorites()` — If `favorites.value.length > 0`, return immediately (frontend cache — backend already has 5-min TTL). Otherwise set `loading = true`, `error = null`, fetch `GET /api/favorites` with `credentials: 'include'`. On success, store in `favorites`, set `loading = false`. On failure (non-ok or network error), set `error = 'Impossible de charger les favoris'`, `loading = false`.
- `reloadFavorites()` — Force-clear `favorites.value = []`, then call `loadFavorites()`. This is for the retry button.
- `playFavorite(zoneUuid: string, favorite: Favorite)` — Fire-and-forget: `POST /api/speakers/${zoneUuid}/play-favorite` with body `{ uri: favorite.uri, type: favorite.type }`, `credentials: 'include'`, `Content-Type: application/json`. No optimistic update needed (WebSocket will push the new state). Silently ignore errors (consistent with sendNext/sendPrevious pattern in zones store).

**Return:** `{ favorites, loading, error, loadFavorites, reloadFavorites, playFavorite }`

Use `import { ref } from 'vue'` and `import { defineStore } from 'pinia'`. No other imports needed.

  <action>
Create `frontend/src/components/FavoritesSheet.vue` using `<script setup lang="ts">` composition API.

**Props:**
```typescript
const props = defineProps<{
  visible: boolean
  zoneName: string
  zoneUuid: string
}>()
const emit = defineEmits<{
  close: []
}>()
```

**Script logic:**
- Import `useFavoritesStore` and `type Favorite` from `@/stores/favorites`
- Import `watch` from `vue`
- `const favStore = useFavoritesStore()`
- Watch `props.visible`: when it becomes `true`, call `favStore.loadFavorites()`
- `handleSelect(fav: Favorite)`: call `favStore.playFavorite(props.zoneUuid, fav)`, then `emit('close')`
- `handleBackdropClick()`: `emit('close')`
- `handleRetry()`: `favStore.reloadFavorites()`

**Template structure (per user decisions — bottom sheet with backdrop):**

```html
<Teleport to="body">
  <Transition name="sheet">
    <div v-if="visible" class="sheet-overlay" @click.self="handleBackdropClick">
      <div class="sheet-panel">
        <!-- Header -->
        <div class="sheet-header">
          <span class="sheet-title">Favoris — {{ zoneName }}</span>
          <button class="sheet-close-btn" @click="emit('close')" aria-label="Fermer">
            <!-- X icon SVG 20x20 -->
          </button>
        </div>

        <!-- Loading skeleton -->
        <div v-if="favStore.loading" class="sheet-body">
          <div v-for="i in 4" :key="i" class="skeleton-line" />
        </div>

        <!-- Error state -->
        <div v-else-if="favStore.error" class="sheet-body sheet-error">
          <p>{{ favStore.error }}</p>
          <button class="retry-btn" @click="handleRetry">Réessayer</button>
```
