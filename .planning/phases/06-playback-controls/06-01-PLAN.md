  <name>Task 1: Add transport command actions to zones store</name>
  <files>frontend/src/stores/zones.ts</files>
  <action>
Add four async action functions to the zones store: `sendPlay(uuid)`, `sendPause(uuid)`, `sendNext(uuid)`, `sendPrevious(uuid)`.

Each action:
1. Reads the current ZoneState from `_zones` Map by UUID
2. Applies optimistic update: for play, set `playState = 'PLAYING'`; for pause, set `playState = 'PAUSED_PLAYBACK'`; for next/previous, no optimistic playState change (track change is async). Trigger reactivity by replacing the Map reference (same pattern as `applyStateChanged`).
3. Calls `fetch('/api/speakers/${uuid}/${command}', { method: 'POST', credentials: 'include' })` — note the `/api/` prefix matching the Vite proxy.
4. On fetch failure (non-ok response or network error): silently revert `playState` to the value saved before the optimistic update. Use the same Map-replace pattern for reactivity.
5. On success: do nothing extra — the WebSocket `state_changed` event will confirm the real state.

Export all four functions from the store's return statement.

Do NOT introduce any new dependencies or imports beyond what's already in the file (ref, computed, defineStore from pinia).
