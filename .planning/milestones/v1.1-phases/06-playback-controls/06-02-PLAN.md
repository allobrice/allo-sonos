  <name>Task 1: Add volume and mute actions to zones store</name>
  <files>frontend/src/stores/zones.ts</files>
  <action>
Add three async action functions to the zones store: `sendVolume(uuid, level)`, `sendMute(uuid)`, `sendUnmute(uuid)`.

**sendVolume(uuid: string, level: number):**
1. Read current ZoneState. Save previous `volume` for revert.
2. Apply optimistic update: set `volume = level`, replace Map reference for reactivity.
3. Call `fetch('/api/speakers/${uuid}/volume', { method: 'PUT', credentials: 'include', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ level }) })`.
4. On failure: silently revert `volume` to saved value, replace Map ref.
5. On success: do nothing (WebSocket confirms).

**sendMute(uuid: string):**
1. Read current ZoneState. Save previous `muted` value.
2. Apply optimistic: set `muted = true`, replace Map ref.
3. Call `fetch('/api/speakers/${uuid}/mute', { method: 'POST', credentials: 'include' })`.
4. On failure: revert `muted`, replace Map ref.

**sendUnmute(uuid: string):**
1. Same pattern as sendMute but set `muted = false` optimistically.
2. Call `fetch('/api/speakers/${uuid}/unmute', { method: 'POST', credentials: 'include' })`.
3. On failure: revert to previous value.

Export all three from the store's return statement (alongside existing exports).

Note: `sendVolume` will be called by the component with debouncing — the store action itself does NOT debounce. It fires immediately for each call.

**Script additions:**

1. Add a `localVolume` ref initialized to `props.zone.volume`. This ref tracks the slider position locally during drag, decoupled from the store.
2. Add a `watch` on `() => props.zone.volume` that updates `localVolume` when the prop changes (WebSocket update), BUT only if the user is not currently dragging. Use a `dragging` ref (boolean) to track this.
3. Add a computed `speakerIcon` that returns the appropriate SVG path string based on: if `zone.muted` -> muted icon; else if `localVolume <= 0` -> muted icon; else if `localVolume <= 33` -> low; else if `localVolume <= 66` -> medium; else -> high.
4. `handleVolumeInput(e: Event)`: called on `@input` (fires continuously during drag). Set `dragging = true`. Update `localVolume.value` to `Number((e.target as HTMLInputElement).value)`. Call debounced volume send.
5. `handleVolumeCommit(e: Event)`: called on `@change` (fires on release). Set `dragging = false`. Send final volume immediately (bypass debounce): `store.sendVolume(props.zone.uuid, Number((e.target as HTMLInputElement).value))`.
6. Debounced volume send: use a simple closure with `setTimeout`/`clearTimeout`. On each `handleVolumeInput`, clear any pending timer and set a new one (250ms delay) that calls `store.sendVolume(props.zone.uuid, localVolume.value)`. This ensures intermediate drags don't flood the API.
7. `handleMuteToggle()`: if `zone.muted`, call `store.sendUnmute(zone.uuid)`; else call `store.sendMute(zone.uuid)`. Use the same `withDebounce` wrapper from 06-01 for anti-double-tap.
