# Plan 05-02: ZoneCard Component

## Goal

Build the `ZoneCard.vue` component and integrate it into the `ZonesView.vue` grid. Each card displays the zone name, now playing (title + artist), the music source icon, and the offline state. All visual decisions from the CONTEXT.md are implemented here.

## must_haves

- All discovered zones appear as cards in a 2-column desktop / 1-column mobile grid — ZONE-01
- Each card shows zone name (visually dominant), track title (bold), and artist (lighter, below) when playing — ZONE-02
- Each card shows a monochrome source icon (top-right corner) for Spotify, Deezer, TuneIn, Library; generic note icon if unknown/null — ZONE-03
- Offline card: grisée (opacity: 0.4), badge "Offline" shown, no now-playing content — ZONE-04
- Active zone (playState === 'PLAYING') has a distinct visual accent (left border or top bar in `--color-accent-green`) — locked decision
- Idle zone (reachable, not playing) shows a muted idle icon + "Aucune lecture" — locked decision
- Offline zone does NOT reorder in the grid (position stays fixed) — locked decision
- Smooth opacity transition when zone comes back online — locked decision
- Truncation with `…` on long title/artist — locked decision
- ZonesView replaces zone-placeholder divs with `<ZoneCard>` components

## Context

**User decisions locked in CONTEXT.md:**
- Card: light shadow (`--shadow-card`), rounded corners (`--radius-md`), background `--color-surface`
- Card height: ~120px (good balance of readability/density)
- Zone name: visually dominant (largest text, primary color)
- Active card accent: colored border or side bar (use left border in `--color-accent-green`)
- Now playing: title bold one line, artist below lighter — truncate with `…`
- Idle: icon + "Aucune lecture" (small muted note icon acceptable)
- Play state indicator: small play/pause icon beside title or zone name
- Source indicator: icon only, top-right, monochrome (white/grey), generic note if unknown
- Offline: opacity reduced (0.4), "Offline" badge, empty except zone name + badge
- Offline transition: CSS `transition: opacity 0.3s ease`
- No marquee/scroll — truncation only

## Tasks

<task id="1" wave="2">
### Create ZoneCard.vue component

Create `frontend/src/components/ZoneCard.vue`. Accept a single `zone: ZoneState` prop.

**Full implementation spec:**

```vue
<script setup lang="ts">
import type { ZoneState } from '@/stores/zones'

defineProps<{ zone: ZoneState }>()

// Source icon mapping — inline SVG paths or Unicode symbols.
// Use simple SVG icons for Spotify, Deezer, TuneIn, Library.
// Fall back to a generic music note for null/unknown source.
function sourceIcon(source: string | null): string {
  // Return a descriptive label used in aria-label; rendering is CSS/template based
  switch (source) {
    case 'spotify': return 'Spotify'
    case 'deezer': return 'Deezer'
    case 'tunein': return 'TuneIn'
    case 'library': return 'Library'
    default: return 'Unknown source'
  }
}
</script>
```

**Card structure (implement in `<template>`):**

```
┌─ .zone-card [active?] [offline?] ──────────────────────┐
│ ╎ (left accent border if active)                        │
│  .zone-header                                           │
│    .zone-name          "Cuisine"                        │
│    .zone-source-icon   [Spotify icon top-right]         │
│                                                         │
│  .zone-body                                             │
│    IF offline:                                          │
│      .offline-badge    "Offline"                        │
│    ELSE IF playing:                                     │
│      .play-icon        ▶ (small)                        │
│      .now-playing                                       │
│        .track-title    "Song Title"  (bold, truncated)  │
│        .track-artist   "Artist Name" (lighter, truncated│
│    ELSE (idle):                                         │
│      .idle-state       ♪ "Aucune lecture"               │
└─────────────────────────────────────────────────────────┘
```

**Computed helpers to use:**
- `isPlaying`: `zone.playState === 'PLAYING'`
- `isOffline`: `!zone.reachable`
- `isActive`: `isPlaying && !isOffline` (for accent)

**CSS requirements:**
- `.zone-card` base: `background: var(--color-surface)`, `border-radius: var(--radius-md)`, `box-shadow: var(--shadow-card)`, `padding: var(--space-md)`, `min-height: 120px`, `position: relative`, `border-left: 3px solid transparent`, `transition: opacity 0.3s ease`
- `.zone-card.active`: `border-left-color: var(--color-accent-green)`
- `.zone-card.offline`: `opacity: 0.4`
- `.zone-name`: `font-size: var(--font-size-lg)`, `font-weight: 600`, `color: var(--color-text-primary)`, `overflow: hidden`, `text-overflow: ellipsis`, `white-space: nowrap`
- `.track-title`: `font-weight: 600`, `overflow: hidden`, `text-overflow: ellipsis`, `white-space: nowrap`, `color: var(--color-text-primary)`, `font-size: var(--font-size-sm)`
- `.track-artist`: `color: var(--color-text-secondary)`, `overflow: hidden`, `text-overflow: ellipsis`, `white-space: nowrap`, `font-size: var(--font-size-sm)`
- `.zone-source-icon`: `position: absolute`, `top: var(--space-sm)`, `right: var(--space-sm)`, `color: var(--color-text-secondary)`, `font-size: var(--font-size-sm)`, `opacity: 0.7`
- `.offline-badge`: `display: inline-block`, `background: var(--color-border)`, `color: var(--color-text-secondary)`, `font-size: var(--font-size-sm)`, `padding: 2px var(--space-sm)`, `border-radius: var(--radius-sm)`
- `.idle-state`: `color: var(--color-text-secondary)`, `font-size: var(--font-size-sm)`, `display: flex`, `align-items: center`, `gap: var(--space-xs)`

**Source icons:** Use Unicode symbols or brief text labels in a `<span>` styled as `.zone-source-icon`. The exact icon implementation is at executor's discretion — acceptable options:
- Unicode: ♫ (generic), or source-specific emoji/symbol
- Short text labels styled small: "SP" / "DZ" / "TI" / "LIB"
- Inline SVG (preferred if concise — keep each icon under 5 lines of SVG)

The executor should choose the most maintainable approach. Monochrome (white/grey) is required.

**Play state icon:** A small `▶` or `⏸` Unicode symbol beside the now-playing section. No icon shown when idle or offline.
</task>
