Create `backend/src/services/sonos-favorites.ts` — the ContentDirectory SOAP service with caching.

**Favorite interface:**
```typescript
export interface Favorite {
  title: string
  type: 'station' | 'playlist' | 'album' | 'other'
  uri: string
  albumArtURI: string | null
}
```

**SOAP Browse function (`soapContentDirectory`):**
- Target URL: `http://${ip}:1400/MediaServer/ContentDirectory/Control`
- SOAPAction: `"urn:schemas-upnp-org:service:ContentDirectory:1#Browse"`
- SOAP envelope body:
  ```xml
  <u:Browse xmlns:u="urn:schemas-upnp-org:service:ContentDirectory:1">
    <ObjectID>FV:2</ObjectID>
    <BrowseFlag>BrowseDirectChildren</BrowseFlag>
    <Filter>*</Filter>
    <StartingIndex>0</StartingIndex>
    <RequestedCount>100</RequestedCount>
    <SortCriteria></SortCriteria>
  </u:Browse>
  ```
- Follow the same fetch() pattern as `soapAvTransport` in `sonos-commands.ts` (raw fetch, not a SOAP library)
- Parse the response XML with `fast-xml-parser` (same `{ removeNSPrefix: true }` config as `sonos-state.ts`)
- The Browse response contains a `Result` field which is a DIDL-Lite XML string (escaped inside the SOAP response) — parse it a second time
- Each `<item>` in DIDL-Lite maps to a Favorite

**Type classification (classifyFavoriteType):**
Map the `upnp:class` value from the DIDL-Lite item metadata to our type:
- `object.item.audioItem.audioBroadcast` → `'station'`
- Anything containing `playlistContainer` or `object.container.playlistContainer` → `'playlist'`
- Anything containing `musicAlbum` or `object.container.album.musicAlbum` → `'album'`
- Any other value → `'other'` (per user decision: no favorites hidden, unknown types exposed as 'other')

**Extracting fields from DIDL-Lite item:**
- `title` → `dc:title` (after NS prefix removal: `title`)
- `uri` → `res` element text content (the Sonos URI like `x-sonosapi-stream:...`)
- `albumArtURI` → `albumArtURI` element text content, or `null` if absent
- `type` → classify from `class` element (was `upnp:class` before NS prefix removal)

**Handle DIDL-Lite parsing edge cases:**
- If `fast-xml-parser` returns a single object instead of an array (when only 1 favorite exists), wrap it in an array
- If `res` is an object with `#text` (parser extracting attributes), use the `#text` value for the URI
- If the DIDL-Lite `Result` is empty or contains no items, return an empty array

**In-memory cache:**
- Module-level variables: `cachedFavorites: Favorite[] | null = null` and `cacheTimestamp: number = 0`
- TTL: 5 minutes (300_000 ms)
- Cache is valid when `cachedFavorites !== null && Date.now() - cacheTimestamp < TTL`

**Public function `fetchFavorites`:**
```typescript
export async function fetchFavorites(speakers: SpeakerInfo[], forceRefresh: boolean = false): Promise<Favorite[]>
```
- If cache is valid and `forceRefresh` is false, return cached data
- Otherwise, pick the first speaker from the `speakers` array (favorites are shared across all Sonos speakers on the network — any reachable one works)
- If no speakers available, return empty array
- Call soapContentDirectory, parse DIDL-Lite, map to Favorite[], cache the result, return it
- On SOAP error, if cache exists (even expired), return stale cache as fallback; otherwise throw

**Route 2: POST /speakers/:id/play-favorite**

Follow the same pattern as the existing play/pause/next routes in `speakers.ts`:
- Params schema: same `paramsSchema` as speakers.ts (`{ id: { type: 'string' } }`)
- Body schema: `{ type: 'object', properties: { uri: { type: 'string' }, type: { type: 'string' } }, required: ['uri', 'type'] }`
- Response schema: same `commandResponseSchema` pattern as speakers.ts (200 with ok+state, 404, 502)

Implementation — replicate the `handleCommand` pattern inline (since handleCommand is not exported from speakers.ts):

```typescript
fastify.post<{ Params: { id: string }; Body: { uri: string; type: string } }>(
  '/speakers/:id/play-favorite',
  { schema: { params: paramsSchema, body: bodySchema, response: commandResponseSchema } },
  async (request, reply) => {
    const { id } = request.params
    const { uri, type } = request.body

    const speaker = fastify.speakers.getById(id)
    if (!speaker) return reply.status(404).send({ error: 'Speaker not found', uuid: id })

    const coordinator = fastify.speakers.getCoordinator(id) ?? speaker

    try {
      // For stations (audio broadcasts): SetAVTransportURI directly
      // For playlists/albums: also use SetAVTransportURI — Sonos handles queue creation internally
      // The metadata is minimal — Sonos resolves it from the URI
      const metadata = ''  // Empty metadata — Sonos resolves from URI
      await soapAvTransport(
        coordinator.ip,
        'SetAVTransportURI',
        `<CurrentURI>${escapeXml(uri)}</CurrentURI><CurrentURIMetaData>${metadata}</CurrentURIMetaData>`,
      )

      // For non-station types, also send Play to start playback
      // (SetAVTransportURI on its own just loads, doesn't auto-play for all types)
      await soapAvTransport(coordinator.ip, 'Play', '<Speed>1</Speed>')

      const state = await readSpeakerState(speaker.ip, coordinator.ip)
      fastify.log.info(`[sonos] play-favorite (${type}) → "${speaker.name}"`)
      return { ok: true, state }
    } catch (err) {
      const detail = err instanceof Error ? err.message : String(err)
      fastify.log.error(`[sonos] play-favorite failed: ${detail}`)
      return reply.status(502).send({ error: 'Command failed', detail })
    }
  },
)
```

**XML escaping helper** (local to this file):
```typescript
function escapeXml(str: string): string {
  return str.replace(/&/g, '&amp;').replace(/</g, '&lt;').replace(/>/g, '&gt;').replace(/"/g, '&quot;').replace(/'/g, '&apos;')
}
```

Note: The `escapeXml` is needed because favorite URIs can contain `&` characters (e.g., `x-sonosapi-stream:s12345?sid=254&flags=8224`).

Export the plugin as default (same pattern as `speakers.ts` and `auth.ts`).
