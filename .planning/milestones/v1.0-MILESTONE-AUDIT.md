### Integration Findings

**Finding 1 — `dotenv: true` diverges from documented decision (MINOR)**
- File: `backend/src/plugins/env.ts`, line 43
- Plan 01-01 SUMMARY documents "dotenv: false" decision
- Impact: Harmless in Docker (no `.env` mounted), but in local dev a `.env` file could silently override `SONOS_PIN`
- Affected: INFRA-01

**Finding 2 — WebSocket messages not consumed in frontend (EXPECTED DEFERRAL)**
- File: `frontend/src/composables/useWebSocket.ts`
- Backend sends `state_changed`, `snapshot`, `speaker_online/offline` events
- Frontend connects but discards all messages; only `connected` ref is used
- Phase 5 will add message parsing and zone state store

**Finding 3 — Playback endpoints have no frontend consumer (EXPECTED DEFERRAL)**
- 8 REST endpoints fully wired in backend, no UI calls them yet
- `ZonesView.vue` is a documented stub placeholder
- Phase 5 will wire zone cards to playback endpoints
